Tính năng 1 — RESUME (Khôi phục bài đang dở):

Tính năng 2 — AUTO-SUBMIT KHI HẾT GIỜ LÚC RESUME:

- Mở lại trang sau khi bài đã hết giờ
- Phải thấy kết quả được nộp tự động (hoặc thông báo hết hạn)

Tính năng 3 — NÚT THOÁT:
- Đang làm bài → bấm ✕ góc trên trái
- Phải thấy hộp xác nhận "Bạn có chắc muốn thoát?"
- Bấm "Ở lại" → tiếp tục làm bài bình thường
- Bấm "Huỷ bài thi" → về hub
- Sau khi huỷ → thi môn khác ngay (không bị 409)

Tính năng 4 — API backend:
- GET /api/exam/active → trả session đang dở (hoặc null)
- POST /api/exam/:id/abandon → 200 { success: true }
- POST /api/exam/:id/abandon với session không thuộc mình → 403
- POST /api/exam/:id/abandon session đã ABANDONED → 409

📋 DEFINITION OF DONE (đối chiếu khi test):
□ GET /api/exam/active → trả về session đang dở kèm remainingSeconds chính xác
□ GET /api/exam/active → trả null khi không có session nào IN_PROGRESS
□ POST /api/exam/:id/abandon → 200, session đổi thành ABANDONED
□ POST /api/exam/:id/abandon của session không thuộc user → 403
□ Sau khi ABANDON → POST /api/exam/start bất kỳ môn → không còn bị 409
□ Vào màn hình thi khi có session dở còn giờ → hiện dialog hỏi tiếp tục
□ Chọn "Tiếp tục": đồng hồ đúng thời gian còn lại, đáp án cũ được khôi phục
□ Chọn "Huỷ" từ dialog: session bị abandon, có thể thi môn khác ngay
□ Vào khi hết giờ: tự động nộp → hiện kết quả (hoặc thông báo hết hạn nếu quá grace)
□ Trong bài: nút "Thoát" → hộp xác nhận → xác nhận → abandon → ra ngoài
□ Không xác nhận thoát → ở lại làm tiếp bình thường
□ Mỗi lần chọn đáp án → tự lưu vào localStorage
