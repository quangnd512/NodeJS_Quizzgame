🎯 TÍNH NĂNG CẦN LÀM: Notifications — Thông báo hệ thống (Feature 013)
🌿 BRANCH: feature/notifications

📝 TÓM TẮT YÊU CẦU NGƯỜI DÙNG:
Thêm hệ thống thông báo tự động cho học sinh. Có 4 loại thông báo:
  1. Streak milestone — học liên tiếp đủ 7/14/30/60/100 ngày
  2. Thay đổi hạng leaderboard — lên hoặc xuống sau khi thi
  3. Báo cáo câu hỏi được admin xử lý — duyệt hoặc từ chối
  4. Đề thi mới active — gửi cho học sinh đã từng học môn đó

Hiển thị 2 cách:
  - Toast popup tự hiện 4 giây khi đang dùng app (polling 30s)
  - Chuông 🔔 ở màn hình chính, badge đỏ số chưa đọc

Bấm vào thông báo:
  - Streak → Trang Tiến độ học tập
  - Lên/xuống hạng → Trang Bảng xếp hạng
  - Báo cáo → chỉ đánh dấu đã đọc, không chuyển trang
  - Đề thi mới → Trang Thi thử

🔧 CHI TIẾT KỸ THUẬT:

DB Schema mới — bảng `notifications`:
  - id: UUID PK
  - userId: FK → users (ON DELETE CASCADE)
  - type: enum (STREAK_MILESTONE, RANK_UP, RANK_DOWN, REPORT_RESOLVED, NEW_EXAM_PAPER)
  - title: TEXT
  - body: TEXT
  - isRead: BOOLEAN default false
  - targetScreen: TEXT nullable (progress | leaderboard | exam | null)
  - metadata: JSON nullable (streakDays, rankBefore/After, reportId, subjectName...)
  - createdAt: DATETIME default now()

Index cần thêm: @@index([userId, isRead, createdAt]) — cho query danh sách nhanh

Files ảnh hưởng (backend):
  - backend/prisma/schema.prisma — thêm model Notification + enum NotificationType
  - backend/src/services/notification/notification.service.ts — file mới
  - backend/src/routes/notification.route.ts — file mới
  - backend/src/app.ts — đăng ký route
  - backend/src/services/practice/practice.service.ts — trigger streak
  - backend/src/services/exam/exam.service.ts — trigger streak + rank change
  - backend/src/routes/admin.route.ts (hoặc service tương ứng) — trigger report resolved
  - backend/src/services/exam/exam.service.ts (createExamPaper/toggleActive) — trigger new paper

Files ảnh hưởng (frontend):
  - frontend/src/lib/api.ts — thêm 4 API calls
  - frontend/src/App.tsx — chuông 🔔 + polling + toast state + NotificationPanel

API draft: docs/api/drafts/notifications.yaml

Edge cases cần xử lý:
  1. Streak milestone: CHỈ tạo thông báo khi streak chạm đúng mốc (7,14,30,60,100), không mỗi ngày. Check sau khi tính streak xong.
  2. Rank change: Lấy rank TRƯỚC khi submit (query leaderboard/me), nộp bài, lấy rank SAU — so sánh. Nếu đổi → thông báo. Dùng fire-and-forget.
  3. New exam paper: Khi admin toggle active=true (không phải lúc tạo) → query userId đã có practice_session hoặc exam_session với subject đó → batch insert notifications. Fire-and-forget.
  4. Tất cả trigger dùng fire-and-forget (void, không await) — không làm chậm response chính.
  5. Polling FE: lưu timestamp lần poll cuối, chỉ hiện toast cho notification mới hơn timestamp đó.

⚠️ LƯU Ý ĐẶC BIỆT:
- Tất cả trigger notification dùng fire-and-forget (void) — không block response chính
- Streak check: chỉ tạo thông báo khi streak value nằm trong [7,14,30,60,100]
- Rank change: cần query rank TRƯỚC và SAU khi submit để so sánh
- New exam paper batch notify: có thể nhiều user → dùng createMany thay vì loop
- Polling frontend: lưu lastCheckedAt, chỉ hiện toast cho notification.createdAt > lastCheckedAt
