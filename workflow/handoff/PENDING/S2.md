LỖ HỔNG CẦN SỬA:

[Bug 1a] Thi thử nộp bài quá sớm:
- Vấn đề: User có thể nộp bài ngay sau khi bắt đầu, không cần làm gì
- Fix: Trong submitExam(), kiểm tra elapsed time >= durationMinutes * 60 * 0.3
- Nếu quá sớm → throw ExamSubmitTooEarlyError (HTTP 400)
- Hằng số: EXAM_MIN_SUBMIT_RATIO = 0.3 (thêm vào exam.types.ts)

[Bug 1b] Lộ đáp án câu bỏ trắng trong kết quả:
- Vấn đề: getExamResult() trả về correctAnswer cho cả câu bỏ trắng
- Fix: Câu bỏ trắng có selectedAnswer = {} (empty object — sentinel "chưa trả lời")
- Nếu selectedAnswer là {} → trả correctAnswer = null trong wrongAnswers
- Cần cập nhật ExamWrongAnswerItem type: correctAnswer: Prisma.JsonValue | null

[Bug 2] Redis tắt → vượt giới hạn luyện tập (fail-open):
- Vấn đề: checkRateLimit() catch lỗi Redis → bỏ qua, cho tạo phiên tiếp
- Fix: Đổi thành fail-closed → khi Redis lỗi → throw PracticeRateLimitError
- File: practice.service.ts, hàm checkRateLimit()

[Bug 3] Phiên luyện tập hết giờ vẫn hoàn thành được:
- Vấn đề: completeSession() không kiểm tra timeout
- Fix: Thêm check elapsed > SESSION_TIMEOUT_SECONDS + 60 giây grace
- Nếu quá hạn → update completedAt + throw PracticeSessionExpiredError
- Kiểm tra phải nằm TRONG transaction

[Bug 4] Mở 2 tab thi thử cùng môn cùng lúc:
- Vấn đề: startExam() không kiểm tra phiên IN_PROGRESS đang tồn tại
- Fix: Trước khi tạo session mới, kiểm tra:
  prisma.examSession.findFirst({ where: { userId, subjectId, status: 'IN_PROGRESS' } })
- Nếu có → throw ExamSessionAlreadyActiveError (HTTP 409)
